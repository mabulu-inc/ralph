## Completion Notes

Exclusion-based task body extraction already implemented in prior iteration. extractTaskBody() uses EXCLUDED_SECTIONS set (Hints, Produces, Completion Notes, Blocked) and includes all other sections. ralph show task T-NNN subcommand displays effective body, hints, excluded sections, and resolved roles with --json support. 45 test files, 1063 tests pass.

## Description

Task body extraction currently uses a whitelist (`extractTaskBody()` includes only Description, Changes, AC, and Scope sections). This means any custom section a user adds (e.g., `## Security Considerations`, `## Migration Plan`) is silently dropped and never reaches the agent. This violates the Task File API contract (§12.2) which states that all sections except a small exclusion list are included.

This task inverts the logic to exclusion-based: include all sections except Hints (sent separately), Produces, Completion Notes, and Blocked. Custom sections automatically flow to the agent.

It also adds the `ralph show task T-NNN` subcommand so users can verify exactly what the agent will receive for a given task.

## Changes

### markdown.ts / tasks.ts

- Change `extractTaskBody()` from whitelist to exclusion-list approach
- Excluded sections: `Hints`, `Produces`, `Completion Notes`, `Blocked`
- All other sections (including user-defined custom sections) are included in the returned body
- Metadata fields (the dashed list at the top) remain excluded as before

### show.ts

- Add `ralph show task T-NNN` subcommand
- Reads the task file, applies the exclusion-based body extraction
- Displays the effective task body that `{{task.description}}` will contain
- Shows the `{{task.hints}}` value separately
- Shows which sections were excluded and why
- Shows the resolved role list for this task (accounting for the `Roles` field if present, plus user customizations from `docs/prompts/roles.md`)
- Supports `--json` flag for structured output
