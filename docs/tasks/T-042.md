# T-042: Externalize all prompt content into user-editable templates

- **Status**: TODO
- **Milestone**: 10 — Prompt Quality & Scalability
- **Depends**: T-041
- **PRD Reference**: §5.3, §5.7, §5.8

## Description

Ensure that every piece of prompt content sent to the agent is fully externalized — no hardcoded prompt text should remain in ralph's source code. All prompt content lives in user-editable template files under `docs/prompts/`, giving teams full control over methodology, tone, and agent behavior.

**Requirements:**

- Audit ralph source for any remaining hardcoded prompt text (error messages to the agent, phase instructions, anti-pattern guidance, etc.)
- Move all prompt content into the template files (`docs/prompts/system.md`, `docs/prompts/boot.md`)
- The only prompt-related logic in ralph source should be: template loading, variable interpolation, and template file path resolution
- `ralph init` must generate all template files with sensible defaults
- Document (in a comment or README within `docs/prompts/`) what each template variable provides, so users can customize effectively
- Add a `ralph init --prompts-only` flag (or similar) to regenerate just the prompt templates without overwriting other project files
- All existing tests must continue to pass

## Hints

- After T-041, system.md and boot.md should contain all prompt content — this task is the final audit and cleanup pass
- Check `loop.ts`, `prompt-template.ts`, and any orchestrator code for string literals that are part of the agent prompt
- Consider adding a `docs/prompts/README.md` that documents available template variables

## Produces

- Updated `docs/prompts/system.md` and `docs/prompts/boot.md` templates
- `docs/prompts/README.md` (or equivalent documentation)
- Updated `src/ralph/commands/init.ts` (--prompts-only flag)
- Audit pass removing any hardcoded prompt text from source
- Tests verifying no prompt content is hardcoded
