## Description

Users need to inspect exactly what the agent sees at runtime — the effective merged content of built-in defaults plus their extensions. PRD §3.9 defines the `ralph show` command with subcommands for each prompt layer.

## Changes

### New: src/ralph/commands/show.ts

Implement the `ralph show` command with subcommands:

- `ralph show system-prompt` — effective system prompt (built-in + user extensions from `docs/prompts/system.md`)
- `ralph show boot-prompt` — effective boot prompt template (built-in + user extensions from `docs/prompts/boot.md`), with template variables shown as placeholders
- `ralph show roles` — all active roles (built-in + custom from `docs/prompts/roles.md`), annotated to show which are built-in, overridden, added, or disabled
- `ralph show methodology` — built-in methodology reference + user extensions from `docs/prompts/methodology.md`
- `ralph show rules` — project-specific rules from `docs/prompts/rules.md`

Options:

- `--json` — output as JSON
- `--built-in-only` — show only ralph's built-in content, ignoring user extensions

### cli.ts

- Register the `show` command with its subcommands

## AC

- Each subcommand displays the merged result of built-in + user extensions
- When no user extensions exist, built-in content is shown as-is
- When user extensions exist, output clearly delineates where built-in ends and extensions begin
- `--json` outputs structured JSON
- `--built-in-only` ignores user extension files
- `ralph show roles` annotates each role with its source (built-in, overridden, added, disabled)
- `ralph show` with no subcommand prints help listing available subcommands
- Existing tests updated, new tests cover each subcommand and flag
- `pnpm check` passes
