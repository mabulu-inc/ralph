## Description

Ralph asks users during init what database they use and stores it in `ralph.config.json`. This was intended to support auto-starting Docker containers before each loop iteration, but that feature was never implemented and the `--no-db` flag was already removed as dead code (T-050). The only surviving use is the `{{config.database}}` template variable which echoes the value into the boot prompt — context the user can provide via `docs/prompts/rules.md` if relevant.

Database and Docker management is the responsibility of the user's test rigging, not ralph. Remove all traces.

## Changes

### PRD (docs/PRD.md)

- §2.2: Remove `Database` from optional config fields
- §3.1: Remove "Database" from the init prompts list
- §3.2: Remove step 2 ("Database — if project has Docker Compose, start containers before each iteration") from the iteration cycle
- §3.2: Remove "Stop database containers (if started)" from cleanup on exit
- §5.1: Remove `{{config.database}}` from the template variables table
- §10: Remove "Ralph does NOT require a database" (no longer relevant if there's no database concept at all)

### ralph.config.json schema (src/ralph/core/config.ts)

- Remove `database` from the config interface and parser
- Ignore the field if present in existing configs (backward compatibility — don't error on unknown fields)

### init.ts

- Remove the database prompt from the interactive init flow

### prompt-template.ts

- Remove `{{config.database}}` from template variable interpolation
- If the variable appears in a user extension file, resolve it to empty string (don't error)

### boot-prompt.ts

- Remove any reference to `{{config.database}}` from the built-in boot prompt template

### Task context for roles (boot prompt)

- Remove the line that uses `{{config.database}}` to signal DBA participation. DBA applicability should be determined from the task content (description mentions data models, schema, queries, etc.), not from a global config flag.

### Default rules.md scaffold (templates/rules-md.ts or init.ts)

Add a hint to the default `rules.md` generated by `ralph init`:

```markdown
<!-- Tip: If your project uses a database, create an early task to set up
     test rigging (e.g., testcontainers, Docker Compose, in-memory SQLite)
     so all subsequent tasks can run integration tests automatically. -->
```
