# T-053: Add README and GitHub Pages documentation site

- **Status**: TODO
- **Milestone**: 12 ‚Äî Documentation
- **Dependencies**: none

## Objective

Create a README.md and a pragmatic, comprehensive GitHub Pages documentation site that helps developers understand the Ralph Methodology and maximize their use of the `@mabulu-inc/ralph` CLI. No fluff or bloat ‚Äî every page should earn its place.

## Requirements

### README.md (project root)

Create a concise, developer-focused README that:

1. **Opens with a one-liner** explaining what ralph is (stateless, PRD-driven AI development loop)
2. **Shows the value prop** ‚Äî why a developer should care (automated TDD, task-driven development, multi-agent support, cost tracking)
3. **Registry setup** ‚Äî explain that `@mabulu-inc/ralph` is published to GitHub Packages and show `.npmrc` configuration:
   ```
   @mabulu-inc:registry=https://npm.pkg.github.com
   //npm.pkg.github.com/:_authToken=${GITHUB_TOKEN}
   ```
   Note: requires a GitHub personal access token with `read:packages` scope.
4. **Quick start** ‚Äî get running in under 60 seconds using `pnpm dlx` or `npx`:
   ```bash
   pnpm dlx @mabulu-inc/ralph init
   # or
   npx @mabulu-inc/ralph init
   ```
5. **Command reference table** ‚Äî all 7 commands (`init`, `loop`, `monitor`, `kill`, `milestones`, `shas`, `cost`) with one-line descriptions
6. **Link to full documentation** ‚Äî points to GitHub Pages site at `https://mabulu-inc.github.io/ralph/`
7. **Requirements** ‚Äî Node.js 20+

### GitHub Pages documentation site (docs-site/)

Follow the same architecture as `simplicity-schema`:

#### Infrastructure

- **Framework**: Astro + @astrojs/starlight
- **Directory**: `docs-site/` at project root (separate from internal `docs/`)
- **Base path**: `/ralph`
- **Site URL**: `https://mabulu-inc.github.io`
- **Package manager**: pnpm
- **Content directory**: `docs-site/src/content/docs/`

#### Site structure and content

Organize documentation into these sections:

1. **Getting Started**
   - `introduction.md` ‚Äî What is Ralph? The methodology in 2 minutes. Why stateless loops beat persistent agents. The benefit: you define tasks, ralph drives your AI agent through red/green TDD automatically.
   - `quick-start.md` ‚Äî Step-by-step from zero to first completed task. Starts with `.npmrc` setup for GitHub Packages (`@simplicity:registry=https://npm.pkg.github.com` + auth token with `read:packages` scope), then `pnpm dlx @mabulu-inc/ralph init`, write a task, run `ralph loop`. Show real terminal output.
   - `configuration.md` ‚Äî `ralph.config.json` fields, supported agents, file naming, quality checks.

2. **Core Concepts**
   - `methodology.md` ‚Äî The Ralph Methodology explained: stateless boot, PRD-driven, task eligibility, red/green/verify cycle, quality gates. Reference `docs/RALPH-METHODOLOGY.md` for source material.
   - `tasks.md` ‚Äî Task file format, status values (TODO/IN-PROGRESS/DONE/BLOCKED/SKIPPED), dependencies, metadata fields, eligibility rules, complexity tiers (Light/Standard/Heavy).
   - `prd.md` ‚Äî How to write an effective PRD for ralph. Structure, inline extraction, how ralph uses it during boot.
   - `prompts.md` ‚Äî Boot prompt templating, the 11 template variables, `docs/prompts/boot.md` and `docs/prompts/rules.md` customization.

3. **Commands**
   - `init.md` ‚Äî Interactive project bootstrapper. What it scaffolds, agent-specific files, prompt templates.
   - `loop.md` ‚Äî The main loop. How it works: boot ‚Üí find eligible task ‚Üí spawn agent ‚Üí red/green/verify ‚Üí commit ‚Üí next. Flags, timeout scaling, retry context injection, codebase indexing.
   - `monitor.md` ‚Äî Real-time progress dashboard. Phase tracking, last output display, activity indicators.
   - `kill.md` ‚Äî Force-stop ralph and child processes. Process tree termination.
   - `milestones.md` ‚Äî Generate milestone summaries with cost breakdowns.
   - `shas.md` ‚Äî Backfill commit SHAs into task metadata.
   - `cost.md` ‚Äî Calculate token usage and costs from JSONL logs.

4. **Agents**
   - `overview.md` ‚Äî Multi-agent support. Provider abstraction pattern. Supported agents: Claude Code, Gemini CLI, Codex CLI, Continue, Cursor.
   - `claude.md` ‚Äî Claude Code setup and configuration.
   - `gemini.md` ‚Äî Gemini CLI setup.
   - `codex.md` ‚Äî Codex CLI setup.
   - `continue.md` ‚Äî Continue setup.
   - `cursor.md` ‚Äî Cursor setup.

5. **Guides**
   - `writing-tasks.md` ‚Äî Best practices for writing effective task files. Dependency graphs, complexity hints, keeping tasks small.
   - `customizing-prompts.md` ‚Äî How to tune boot prompts and rules for your project. Template variable reference.

#### Tone and style

- Direct, declarative language. No marketing speak.
- Code-first: every concept gets a real example.
- Progressive disclosure: overview ‚Üí quick start ‚Üí reference ‚Üí deep-dive.
- Use Starlight callout boxes (`::: tip`, `::: note`, `::: caution`) sparingly and only when genuinely useful.

- Social link to GitHub repo

- **The methodology is the product** ‚Äî help developers understand the stateless, PRD-driven approach and why it produces better results than ad-hoc AI prompting
