## Description

The monitor's per-phase durations and live timer (T-047, T-049) depend on each JSONL log line having a `timestamp` field. However, `spawnWithCapture` in `src/ralph/core/process.ts` pipes raw agent stdout directly to the log file — no timestamps are injected. As a result, `extractTimestampFromJsonLine()` always returns `null` and all timing features silently degrade to showing no durations.

**Requirements:**

- Modify `spawnWithCapture` (or introduce a transform stream) so that each complete JSONL line written to the log file is enriched with a `"timestamp": "<ISO 8601>"` field reflecting when the line was received
- Buffer incoming data to handle partial lines (the agent may flush mid-line)
- Non-JSON lines (e.g., stderr output) should be written through unchanged
- Existing tests for `spawnWithCapture`, `monitorProcess`, and the monitor's phase parsing must continue to pass
- Add tests that verify timestamp injection on captured JSONL output

## Touches

## Hints

- Currently line 39 does `child.stdout?.pipe(stream)` — replace with a Transform stream or line-buffered listener that parses each line, injects the field, and writes
- Use `readline` or a simple newline-split buffer — each JSONL line ends with `\n`
- `JSON.parse` → add `timestamp` key → `JSON.stringify` → write + `\n`
- Stderr is also piped (line 40) — leave it as-is or wrap similarly, but stderr is not JSONL so don't try to parse it
- The monitor's `extractTimestampFromJsonLine()` already looks for `obj.timestamp` as a string — once injected, all existing monitor code will work without changes
