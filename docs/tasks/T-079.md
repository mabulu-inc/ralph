# T-079: Methodology extension via docs/prompts/methodology.md

- **Status**: TODO
- **Milestone**: 15 — Prompt Architecture
- **Depends**: T-076
- **PRD Reference**: §5, §5.8
- **Complexity**: light
- **Touches**: `src/ralph/core/prompt-template.ts`, `src/ralph/templates/methodology.ts`

## Description

The Ralph Methodology reference is currently copied into the user's project as `docs/RALPH-METHODOLOGY.md` by `ralph init`. Under the built-in-first architecture (T-076), the methodology lives in ralph's code and is injected at runtime. Users extend it via an optional `docs/prompts/methodology.md` file.

This task wires the methodology into the built-in-first prompt assembly and removes it from `ralph init` file generation.

## Changes

### prompt-template.ts

- Import the methodology template from `src/ralph/templates/methodology.ts`
- Include the built-in methodology content in the system prompt layer
- Check for `docs/prompts/methodology.md` in the user's project — if it exists, append its content after the built-in methodology

### init.ts

- Stop generating `docs/RALPH-METHODOLOGY.md` during `ralph init` (already handled by T-076, but verify)

## AC

- The built-in methodology is included in the effective system prompt at runtime
- If `docs/prompts/methodology.md` exists, its content is appended to the built-in methodology
- If `docs/prompts/methodology.md` does not exist, the built-in methodology is used alone
- `ralph show methodology` displays the effective merged content
- Existing tests updated, new tests cover methodology extension
- `pnpm check` passes
