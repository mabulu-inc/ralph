## Description

`ralph init` defaults `qualityCheck` to `${packageManager} check` and `testCommand` to `${packageManager} test`. When `packageManager` is `npm`, this produces `npm check` which fails because `npm` requires `run` for custom scripts (unlike `pnpm`/`yarn`). The defaults should be `npm run check` and `npm run test` when npm is selected.

GitHub Issue: #1

## AC

- When `packageManager` is `npm`, the default `qualityCheck` is `npm run check` (not `npm check`)
- When `packageManager` is `npm`, the default `testCommand` is `npm run test` (not `npm test` — consistent with qualityCheck even though `npm test` works as a built-in alias)
- When `packageManager` is `pnpm` or `yarn`, defaults remain unchanged (`pnpm check`, `pnpm test`, etc.)
- Existing tests pass, new tests confirm npm defaults include `run`
- `pnpm check` passes
- Comment on GitHub issue #1 confirming the fix and close the issue
