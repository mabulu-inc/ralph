# T-073: Extract full task body for boot prompt

- **Status**: TODO
- **Milestone**: 13 ‚Äî Prompt Fidelity
- **Depends**: none

- **Complexity**: standard

## Description

`task.description` currently uses `extractSectionFirstParagraph`, which returns only the first paragraph under `## Description`. Task files commonly include additional sections like `## Changes`, `## AC`, and `## Scope` that contain critical acceptance criteria and implementation details the agent needs to follow. These sections are silently dropped, causing the agent to skip work items it was never told about.

The fix should extract the full body of the task file (everything the agent needs to act on) and expose it as `{{task.description}}` ‚Äî or as a new variable if backward compatibility requires it. At minimum, `## Description`, `## Changes`, `## AC`, and `## Scope` content must reach the agent. The `## Hints` section is already handled separately via `{{task.hints}}` and should not be duplicated. Metadata fields (Status, Milestone, Depends, etc.), `## Produces`, `## Completion Notes`, and `## Blocked` sections are not actionable and should be excluded.

## AC

- The agent receives the full content of `## Description` (not just the first paragraph)
- Content from `## Changes`, `## AC`, and `## Scope` sections reaches the agent in the boot prompt
- `## Hints` is not duplicated (already in `{{task.hints}}`)
- Non-actionable sections (`## Produces`, `## Completion Notes`, `## Blocked`, metadata fields) are excluded
- The prompts-readme template description matches the actual behavior
- Existing tests updated, new tests cover multi-paragraph and multi-section extraction
- `pnpm check` passes
