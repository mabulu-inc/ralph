# T-058: Parse explicit Complexity field from task files

- **Status**: DONE
- **Milestone**: 8 — Robustness
- **Depends**: none
- **PRD Reference**: §3.4
- **Completed**: 2026-03-16

## Completion Notes

Added optional `complexity` field to the `Task` interface, parsed from `- **Complexity**: light|standard|heavy` metadata in task files. `scaleForComplexity` now checks `task.complexity` first and falls back to the heuristic estimator only when absent. Orchestrator log line shows `[task-file]` or `[heuristic]` source. Invalid complexity values are silently ignored (treated as absent). All 819 existing tests pass plus 7 new tests covering parsing and scaling behavior.

## Description

The current regex/heuristic complexity estimator (`src/ralph/core/complexity.ts`) misclassifies tasks because it relies on keyword matching and counting produces/dependencies. A task like "Test infrastructure — Docker lifecycle and per-test database isolation" that touches integration tests across 6 packages scored `standard` (75 turns) instead of `heavy` (125 turns) and ran out of turns at 76.

Task files can now include a `- **Complexity**: light|standard|heavy` metadata field (documented in RALPH-METHODOLOGY.md). The orchestrator should read this field and use it directly, falling back to the heuristic only when the field is absent.

## Requirements

1. **Add a `Complexity` metadata field to the task parser** — `src/ralph/core/tasks.ts` should parse `- **Complexity**: light|standard|heavy` from task files into a new optional `complexity` field on the `Task` interface.

2. **Prefer explicit over heuristic** — `scaleForComplexity` should check `task.complexity` first. If it's a valid tier, use it directly. Otherwise fall back to `computeTaskComplexity`.

3. **Log the source** — when the orchestrator logs the complexity tier (see T-057), indicate whether it came from the task file or the heuristic fallback.

4. **All existing tests must continue to pass.**

## Produces

- Updated `Task` interface with optional `complexity` field
- Updated task parser to extract `Complexity` from metadata
- Updated `scaleForComplexity` to prefer explicit tier over heuristic
- Tests for parsing and fallback behavior

## Hints

- Valid tiers are `light`, `standard`, `heavy` — reject anything else and fall back to heuristic
