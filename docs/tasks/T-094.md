## Description

`ralph review T-NNN --coach` fails with "Coaching requires an agent configuration" even when `ralph.config.json` has `"agent": "claude"` configured. The loop command reads config via `readConfig()` and uses the agent field — the review command should do the same but doesn't.

The `run()` function in review.ts expects `coachOpts` as an optional parameter passed by the caller, but the CLI entry point (`bin.ts`) calls `run(args)` without reading the config or constructing coaching options. The config is sitting right there in `ralph.config.json` — the review command just needs to read it the same way the loop does.

## Changes

### review.ts

- When `--coach` is detected and no `coachOpts` were passed by the caller, read `ralph.config.json` via the existing `readConfig()` function (same one the loop uses)
- Construct `CoachingOptions` from the config's `agent` field
- Build the `spawnAgent` callback using the agent provider (same pattern the loop uses to spawn agent sessions)
- Support `--agent <name>` as a CLI override, same as `ralph loop --agent`
- Only error if neither the config nor `--agent` provides an agent

## AC

- `ralph review T-NNN --coach` works when `ralph.config.json` has an `agent` field, without passing `--agent`
- `ralph review --coach --agent claude` works as a CLI override
- If neither config nor `--agent` provides an agent, the error message is shown
- Existing tests updated, new tests cover config-based agent resolution
- `pnpm check` passes
