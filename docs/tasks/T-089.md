# T-089: Add --version flag to CLI

- **Status**: TODO
- **Milestone**: 8 — Robustness
- **Depends**: none
- **PRD Reference**: §3
- **Complexity**: light
- **Touches**: `src/ralph/cli.ts`, `src/ralph/bin.ts`, `README.md`, `docs-site/src/content/docs/**`

## Description

`ralph --version` and `ralph -V` should print the package version and exit. Currently these are treated as unknown commands. Users need this to verify which version of ralph is running, especially when using `pnpm dlx` which always fetches the latest.

## Changes

### cli.ts

- Add `'version'` to `DispatchResult` type
- In `dispatch()`, check for `--version` or `-V` as the first argument and return `{ action: 'version' }`

### bin.ts

- Handle the `version` action by reading `version` from `package.json` and printing it to stdout

### README.md

- Add `--version` to the usage/commands section so users know how to check the version

### docs-site

- Update the Getting Started or Commands overview page to mention `ralph --version`
- If a CLI reference page exists, add `--version` / `-V` there

## AC

- `ralph --version` prints the version number (e.g., `0.7.0`) and exits with code 0
- `ralph -V` does the same
- Existing `--help` behavior unchanged
- README documents the flag
- Docs site mentions the flag
- New tests cover both flags
- `pnpm check` passes
